# Color Guide
## Composable Business Application Platform (CBAP)

This document defines the color system for CBAP's auto-generated UX. The platform uses soft, pastel color palettes to create a calm, professional interface suitable for business applications.

---

## Design Principles

1. **Soft & Pastel**: All colors use muted, pastel tones to reduce eye strain during long work sessions
2. **Accessibility First**: All color combinations meet WCAG 2.1 AA contrast requirements
3. **Semantic Clarity**: Colors convey meaning (success, warning, error, info) without being harsh
4. **Theme Consistency**: Dark and light themes maintain the same semantic relationships
5. **Professional**: Colors support a business application context, not entertainment

---

## Color Schemes

The platform supports two color schemes:

- **Light Theme**: Default theme for standard office environments
- **Dark Theme**: Alternative theme for low-light environments and user preference

## Accessibility Guidelines

### Contrast Ratios

All color combinations must meet WCAG 2.1 AA standards:

- **Normal Text**: Minimum 4.5:1 contrast ratio
- **Large Text** (18pt+ or 14pt+ bold): Minimum 3:1 contrast ratio
- **Interactive Elements**: Minimum 3:1 contrast ratio
- **UI Components**: Minimum 3:1 contrast ratio

### Color Independence

- **Never rely solely on color** to convey information
- Always pair color with icons, text labels, or patterns
- Ensure information is accessible in grayscale

### Focus Indicators

- All interactive elements must have visible focus indicators
- Focus rings use Primary color with 2px width
- Focus indicators must have at least 3:1 contrast ratio

## Implementation Notes

### CSS Variables

The color system should be implemented using CSS custom properties (variables) to enable theme switching:

```css
:root[data-theme="light"] {
  --color-bg-primary: #FAF9F7;
  --color-text-primary: #2C2A28;
  --color-primary: #7B9BC8;
  /* ... */
}

:root[data-theme="dark"] {
  --color-bg-primary: #1E1D1B;
  --color-text-primary: #F5F3F0;
  --color-primary: #8BB5D1;
  /* ... */
}
```

### Theme Switching

- Users should be able to switch themes via user preferences
- System preference (prefers-color-scheme) should be respected on first load
- Theme preference should be persisted in user settings

### Color Usage Rules

1. **Consistency**: Use semantic colors consistently across all components
2. **Moderation**: Avoid overusing bright colors; maintain a calm, professional appearance
3. **Hierarchy**: Use color to establish visual hierarchy, not decoration
4. **Context**: Consider the context when applying colors (e.g., error states in forms vs. error messages)
